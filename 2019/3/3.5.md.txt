#  如何验证高可用  混沌工程 FMEA 与循证医疗在IT系统中的应用 

  [不可用的原因](https://en.wikipedia.org/wiki/High_availability#cite_note-15)
  
  2010年学术可用性专家的一项调查分析了企业IT系统不可用的原因。所有原因均指未遵循以下每个方面的最佳做法（按重要性排序）：[13]
  
  + 监控相关组件
  + 要求和采购
  + 操作
  + 避免网络故障
  + 避免内部应用程序故障
  + 避免失败的外部服务
  + 物理环境
  + 网络冗余
  + 备份技术方案
  + 备份的处理解决方案
  + 物理位置
  + 基础设施冗余
  + 存储架构冗余
